# Contributing

Thank you for wanting to contribute to this knowledge source! Your contributions help everyone learn and improve.

What we welcome
- Corrections, clarifications, and improvements to existing notes.
- New notes, examples, or worked problems that fit the repository's scope.
- Formatting, organization, and link fixes to improve readability.

How to contribute (quick)
1. Fork the repository (or branch directly if you have permissions).
2. Create a feature branch: `git checkout -b fix/your-change`
3. Make small, focused changes and add tests/examples if relevant.
4. Commit with a clear message and push the branch.
5. Open a Pull Request describing what you changed and why.

PR guidelines
- Explain the goal of the change in the PR description.
- Reference any related issues using `#<issue-number>`.
- If the change affects examples or code, include a short verification step or example.
- Keep PRs small and focused when possible.

Reporting issues
- Use the Issues tab to report missing topics, errors, or requests for new notes.
- Provide steps to reproduce (if applicable), the expected result, and what you saw.

Style / formatting
- Prefer clear, simple language and short sections.
- Use consistent headings, code fences for commands and code, and relative links to other files.
- If adding code, include a minimal example and expected output.

Communication
- Be respectful and patient — contributions may be reviewed before merging.
- If your PR sits idle, feel free to politely ask for a review.

License and attribution
- By contributing, you agree to license your contribution under the repository's license (see LICENSE).
- If you add content based on another source, please add attribution and a link.

Thanks again — every contribution helps the whole community!
